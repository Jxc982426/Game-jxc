## 📋 修订概述

本文档说明了根据用户需求将原 3D 架构文档修改为纯 2D 架构的所有变更内容。

**修订日期：** 2026-04-02

**适用引擎：** Godot 4.x

## 一、主要变更内容

### 1.1 节点类型变更

| 原 3D 节点 | 新 2D 节点 | 说明 |
|-----------|-----------|------|
| Node | Node2D | 所有场景根节点改为 2D |
| CharacterBody3D | CharacterBody2D | 角色刚体从 3D 改为 2D |
| Camera3D | Camera2D | 摄像机从 3D 改为 2D |
| MeshInstance3D | Sprite/Sprite2D | 3D 模型改为 2D 精灵 |
| AnimationPlayer | AnimatedSprite2D | 动画播放器改为 2D 动画精灵 |
| NavigationAgent3D | NavigationAgent2D | 3D 寻路改为 2D 寻路 |
| NavigationRegion3D | NavigationRegion2D | 3D 导航网格改为 2D |
| CollisionShape3D | CollisionShape2D | 3D 碰撞体改为 2D |
| Area3D | Area2D | 3D 区域检测改为 2D |
| RayCast3D | RayCast2D | 3D 射线改为 2D |
| Marker3D | Marker2D | 3D 标记点改为 2D |
| GPUParticles3D | GPUParticles2D | 3D 粒子改为 2D 粒子 |
| DirectionalLight3D | DirectionalLight2D | 3D 平行光改为 2D |
| PointLight3D | PointLight2D | 3D 点光源改为 2D |
| AudioStreamPlayer3D | AudioStreamPlayer | 3D 音频改为普通音频 |
| WorldEnvironment | 删除 | 2D 不需要世界环境设置 |
| Decal | 删除 | 2D 不支持贴花投影 |

---

### 1.2 数据结构变更

| 原 3D 类型 | 新 2D 类型 | 说明 |
|-----------|-----------|------|
| Vector3 | Vector2 | 3D 向量改为 2D 向量 |
| Transform3D | Transform2D | 3D 变换改为 2D |
| Basis | 无 | 2D 不需要基向量 |
| Quaternion | float | 3D 四元数改为 2D 角度 |

---

### 1.3 资源类型变更

| 原 3D 资源 | 新 2D 资源 | 说明 |
|-----------|-----------|------|
| Texture | Texture2D | 纹理贴图明确为 2D |
| PackedScene (3D 模型) | SpriteFrames | 3D 模型包改为 2D 动画帧 |
| NavigationMesh | NavigationPolygon | 3D 导航网格改为 2D 多边形 |
| StandardMaterial3D | StandardMaterial2D / CanvasItemMaterial | 3D 材质改为 2D |

---

### 1.4 目录结构变更

```
原 3D 版本:
resources/
├── textures/
├── models/          # 3D 模型

新 2D 版本:
resources/
├── sprites/         # 2D 精灵图
│   ├── characters/
│   ├── enemies/
│   ├── items/
│   └── tiles/
├── animations/      # 2D 动画资源
```

---

### 1.5 坐标系统变更

**3D 版本:**
- 使用 XYZ 三轴坐标
- Y 轴通常向上
- 移动需要三个维度

**2D 版本:**
- 使用 XY 两轴坐标
- Y 轴向下（Godot 2D 惯例）
- 移动只需要两个维度
- 更简单的碰撞检测和物理计算

---

### 1.6 光照系统变更

**3D 版本:**
- WorldEnvironment 全局环境光
- DirectionalLight3D 模拟太阳
- PointLight3D 点光源
- 完整的 3D 光照和阴影

**2D 版本:**
- 简化的 2D 光照系统
- PointLight2D 用于物品发光
- DirectionalLight2D 可选
- OccluderPolygon2D 创建 2D 阴影
- 更多依赖 Sprite 本身的光照效果

---

### 1.7 粒子系统变更

**3D 版本:**
- GPUParticles3D
- 3D 空间发射器
- 复杂的 3D 运动轨迹

**2D 版本:**
- GPUParticles2D
- 2D 平面发射器
- 简化的 2D 粒子运动
- 更适合俯视角游戏

## 四、性能考虑差异

### 4.1 3D vs 2D 性能对比

**3D 优势:**
- 视觉效果更真实
- 可以旋转视角
- 适合复杂地形

**2D 优势:**
- 性能开销更小
- 开发周期更短
- 美术资源更便宜
- 更适合挂机类游戏
- 移动端兼容性更好

---

### 4.2 2D 优化建议

```
1. 合批处理（Batching）
   - Godot 自动合并相同材质的 2D 绘制调用
   - 使用 TextureAtlas 减少材质切换

2. 视口剔除（Visibility Culling）
   - 只渲染摄像机范围内的物体
   - 使用 VisibleOnScreenNotifier2D

3. 粒子系统优化
   - 限制同时存在的 2D 粒子数量
   - 使用 GPU 粒子而非 CPU 粒子

4. 精灵优化
   - 使用合适的纹理压缩格式
   - 复用精灵帧（Animation 共享 SpriteFrames）

5. 碰撞优化
   - 使用简单的碰撞形状（圆形/矩形）
   - 分层碰撞检测
```

## 六、后续建议

### 6.1 推荐的美术风格

1. **像素风格**
   - 开发成本低
   - 怀旧感强
   - 适合独立游戏

2. **手绘风格**
   - 视觉效果精美
   - 成本较高
   - 适合高品质游戏

3. **Low Poly 2D**
   - 简约风格
   - 性能优秀
   - 独特视觉体验

---

### 6.2 推荐的分辨率

**标准比例：** 16:9

**常见分辨率:**

- 1920×1080 (Full HD)
- 1280×720 (HD)
- 960×540 (qHD, 推荐用于挂机游戏)

**Godot 设置:**

```
Display > Window > Size:
- Viewport Width: 960
- Viewport Height: 540
- Stretch Mode: canvas_items
- Aspect: keep
```
