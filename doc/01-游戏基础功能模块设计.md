# 01-暗黑挂机刷刷刷小游戏 - 基础功能模块设计

## 📋 文档概述

本文档描述了一款参考《暗黑 2》的可挂机刷刷刷小游戏的基础功能模块设计。该游戏旨在提供轻松的游戏体验，玩家可以通过自动挂机系统获得游戏乐趣和成就感。

**游戏类型：** 2D 俯视角 ARPG
**开发引擎：** Godot 4.x
**美术风格：** 2D 像素/手绘风格

---

## 一、核心战斗系统

### 1. 角色控制系统

**功能描述：**
- **角色移动**：支持点击地面移动和键盘（WASD/方向键）控制两种模式
- **基础攻击**：根据职业区分近战攻击、远程攻击或技能攻击
- **自动锁定敌人**：自动选择范围内最近的敌人作为目标
- **碰撞检测与命中判定**：确保攻击与移动的准确性

**实现要点：**
- 使用 Godot 的 CharacterBody2D 作为角色基类
- 实现状态机管理（Idle/Moving/Attacking/Damaged）
- 鼠标点击使用 NavigationServer2D 进行 2D 寻路
- 攻击判定使用 Area2D 检测碰撞
- Sprite/AnimatedSprite2D 负责 2D 渲染

---

### 2. 敌人 AI 系统

**功能描述：**
- **基础巡逻逻辑**：敌人在指定区域内随机巡逻
- **追击玩家逻辑**：当玩家进入仇恨范围时开始追击
- **攻击行为**：根据怪物类型执行近战、远程或技能攻击
- **仇恨值管理**：多玩家场景下的目标选择机制

**实现要点：**
- 使用有限状态机（FSM）实现 AI 状态切换
- 状态包括：Idle（巡逻）、Chase（追击）、Attack（攻击）、Return（返回）
- 使用 NavigationAgent2D 进行 2D 路径追踪
- 仇恨列表记录伤害来源和数值

---

### 3. 战斗计算系统

**功能描述：**
- **伤害计算公式**：支持物理伤害、魔法伤害、元素伤害等多种类型
- **暴击/闪避/格挡机制**：增加战斗的随机性和策略性
- **护甲与抗性减免**：防御属性的减伤计算
- **攻击速度与施法速度**：影响输出频率

**核心公式参考：**
```
最终伤害 = 基础伤害 × (1 + 攻击力/防御力系数) × 暴击倍率 × 元素加成
物理减伤 = 护甲 / (护甲 + K)
元素减伤 = 抗性百分比
实际伤害 = 原始伤害 × (1 - 物理减伤) × (1 - 元素减伤)
```

---

## 二、角色成长系统

### 4. 属性系统

**功能描述：**
- **基础属性**：力量（增加物理攻击）、敏捷（增加攻速和闪避）、智力（增加法术强度和法力值）、体力（增加生命值）
- **衍生属性**：生命值、法力值、攻击力、防御力、暴击率、闪避率等
- **属性点分配机制**：升级获得属性点，自由分配到基础属性

**数据结构示例：**
```
基础属性 {
    力量：int
    敏捷：int
    智力：int
    体力：int
}

衍生属性 {
    最大生命值：float
    最大法力值：float
    物理攻击力：float
    法术强度：float
    护甲值：float
    暴击率：float
    闪避率：float
}
```

---

### 5. 等级系统

**功能描述：**
- **经验值获取与升级**：击杀怪物、完成任务获得经验值
- **每级属性成长**：升级后自动增加基础属性
- **升级奖励**：获得属性点和技能点用于角色强化

**经验曲线设计：**
```
升级所需经验 = 基础经验 × (等级 ^ 经验指数)
推荐参数：基础经验=100, 经验指数=1.5
```

---

### 6. 技能系统

**功能描述：**
- **技能树设计**：每个职业拥有独立的技能树，分为多个分支
- **技能学习与升级**：消耗技能点学习或升级技能
- **技能效果**：包括伤害技能、BUFF 技能、DEBUFF 技能、召唤技能等
- **法力消耗与冷却时间**：平衡技能使用频率

**技能分类：**
- **主动技能**：需要手动释放，有冷却时间和法力消耗
- **被动技能**：永久生效，提供属性加成或特殊效果
- **终极技能**：强力技能，需要达到特定等级学习

---

## 三、装备系统

### 7. 装备槽位管理

**功能描述：**
- **装备部位**：头部、衣服、武器（主手/副手）、项链、戒指×2、鞋子、手套、腰带
- **装备穿戴与卸下**：拖拽或点击方式装备物品
- **装备切换**：支持多套装备方案快速切换（如挂机装、BOSS 装）

**装备槽位限制：**
- 每个部位只能装备一件同类型装备
- 装备需要满足等级和属性要求才能穿戴

---

### 8. 装备属性生成

**功能描述：**
- **品质分级**：普通（白色）、魔法（蓝色）、稀有（黄色）、传奇（橙色）
- **随机词缀系统**：装备生成时随机附加属性词条
- **固定属性与随机属性**：基础属性固定，额外属性随机
- **装备等级需求**：不同等级的装备有不同的属性区间

**品质概率参考：**
```
普通：60%
魔法：30%
稀有：9%
传奇：1%
```

**词缀示例：**
- 前缀："+5 力量"、"+10% 暴击伤害"
- 后缀："的生命值"、"的火焰抗性"

---

### 9. 装备掉落系统

**功能描述：**
- **掉落表配置**：每个怪物/BOSS 有独立的掉落池
- **掉落概率控制**：根据怪物难度和品质调整掉率
- **品质颜色显示**：不同品质装备有不同颜色的光柱特效
- **自动拾取/手动拾取**：可配置自动拾取品质和数量限制

**掉落流程：**
1. 怪物死亡时触发掉落判定
2. 根据掉落表确定是否掉落物品
3. 生成物品并随机化属性
4. 显示掉落光柱和提示
5. 玩家拾取或自动收入背包

---

## 四、物品与经济系统

### 10. 背包系统

**功能描述：**
- **背包格子管理**：初始格子数量有限，可扩展
- **物品堆叠**：消耗品和材料可以堆叠，装备不可堆叠
- **物品分类**：装备、消耗品（药水/卷轴）、材料（打造/合成）、任务物品
- **背包整理与筛选**：按品质、类型排序，快速查找物品

**背包扩展方式：**
- 金币购买格子
- 完成任务奖励背包
- VIP 特权增加格子

---

### 11. 商店系统

**功能描述：**
- **NPC 商店买卖**：与商人 NPC 交互打开商店界面
- **物品价格计算**：基于物品品质和属性动态定价
- **刷新机制**：定时刷新商店商品或使用刷新道具

**买卖规则：**
```
出售价格 = 基础价格 × 品质系数 × 0.5（半价回收）
购买价格 = 基础价格 × 品质系数
```

---

### 12. 金币系统

**功能描述：**
- **金币获取**：击杀怪物掉落、任务奖励、出售物品
- **金币消耗**：购买装备/消耗品、修理装备、合成打造、扩展背包

**经济平衡要点：**
- 控制金币产出速度，避免通货膨胀
- 设置足够的金币消耗途径
- 后期可引入绑定金币和非绑定金币双货币系统

---

## 五、挂机系统 ⭐核心特色

### 13. 自动战斗模块

**功能描述：**
- **自动寻路到怪物点**：根据配置的路径点自动移动
- **自动选择目标**：优先选择最近或最低血量的敌人
- **自动使用技能**：根据配置的优先级释放技能
- **自动喝药补给**：血量/蓝量低于阈值时自动使用药水

**挂机状态机：**
```
寻找目标 → 移动到目标 → 进入攻击范围 → 释放技能 → 目标死亡 → 寻找下一个目标
         ↓
      血量不足 → 喝药/撤退
```

---

### 14. 挂机策略配置

**功能描述：**
- **技能释放优先级**：设置技能的使用顺序和条件
- **血量/蓝量阈值设置**：自定义喝药和撤退的血量线
- **挂机区域选择**：选择适合的挂机场地图和点位
- **拾取过滤规则**：只捡取有价值物品的配置

**策略配置界面：**
- 技能栏拖拽排序设置优先级
- 滑动条设置血量/蓝量百分比阈值
- 勾选框设置拾取品质（如只捡稀有及以上）
- 下拉框选择攻击偏好（最近/最低血量/最高威胁）

---

### 15. 离线收益模块（可选）

**功能描述：**
- **离线时长计算**：记录玩家下线时间和上线时间差
- **离线收益结算**：根据挂机效率和时长计算收益
- **收益上限控制**：防止离线收益过高影响平衡

**收益计算公式：**
```
离线收益 = 挂机效率 × 离线时长 × 衰减系数
最大离线时长：建议 8-12 小时
衰减系数：超过 4 小时后每小时递减 10%
```

---

## 六、地图与场景系统

### 16. 地图生成系统

**功能描述：**
- **固定地图设计**：精心设计的关卡布局和怪物分布
- **或随机地图生成**：Roguelike 元素，每次进入地图布局不同
- **地图传送点/Waypoint**：已激活的传送点可快速移动

**地图要素：**
- 出生点：玩家进入地图的位置
- 怪物刷新点：定义怪物类型和数量
- 资源点：宝箱、矿脉、草药等
- BOSS 房间：地图终点的强力怪物
- 传送门：通往下一层或返回城镇

---

### 17. 场景交互

**功能描述：**
- **可破坏物体**：箱子、罐子、木桶等可打碎物品
- **机关与陷阱**：需要触发或躲避的环境要素
- **地形障碍**：墙壁、河流、悬崖等阻挡地形

**可破坏物体掉落：**
- 金币（100% 掉落）
- 药水（小概率）
- 装备（极低概率）

---

## 七、任务与进度系统

### 18. 任务系统

**功能描述：**
- **主线任务推进**：引导玩家体验剧情和解锁功能
- **支线任务**：提供额外奖励和背景故事
- **每日任务**：每天刷新，增加玩家留存和活跃度

**任务类型：**
- 击杀任务：消灭指定数量的怪物
- 收集任务：收集特定物品
- 探索任务：发现隐藏区域或传送点
- BOSS 任务：击败特定 BOSS

---

### 19. 成就系统

**功能描述：**
- **击杀成就**：累计击杀一定数量的怪物或精英
- **收集成就**：收集全套装备或稀有物品
- **挑战成就**：完成高难度挑战（如无伤通关、限时通关）

**成就奖励：**
- 称号（展示用）
- 属性点/技能点重置券
- 稀有外观/坐骑
- 成就点数（可兑换奖励）

---

## 八、数据与存档系统

### 20. 存档系统

**功能描述：**
- **角色数据保存**：等级、属性、技能点等
- **背包数据保存**：所有物品及其属性
- **任务进度保存**：已完成/进行中的任务
- **自动存档/手动存档**：定期自动保存 + 关键节点手动保存

**存档时机：**
- 进入游戏/退出游戏
- 完成任务
- 更换装备
- 到达新地图
- 定时保存（每 5 分钟）

---

### 21. 配置表系统

**功能描述：**
- **怪物配置表**：定义怪物的属性、技能、掉落
- **装备配置表**：定义装备的基础属性和词缀池
- **技能配置表**：定义技能的效果、消耗、冷却
- **掉落配置表**：定义掉落概率和掉落池

**配置格式建议：**
- 使用 JSON 或 CSV 格式
- 支持热更新（不重启游戏加载新配置）
- 使用 ID 引用避免硬编码

---

## 九、UI 界面系统

### 22. 基础 UI

**功能描述：**
- **角色信息面板**：显示头像、等级、经验条、血条、蓝条
- **背包界面**：网格状展示物品，支持拖拽操作
- **角色面板**：详细展示属性、装备槽位、抗性等信息
- **技能栏**：快捷施法按键，显示冷却和法力消耗

**UI 布局建议：**
```
┌─────────────────────────────┐
│ 小地图    │   角色信息     │
├───────────┤                │
│           │                │
│  游戏画面  │   聊天/日志    │
│           │                │
│           ├────────────────┤
│           │   技能栏       │
└───────────┴────────────────┘
```

---

### 23. 交互 UI

**功能描述：**
- **对话框**：与 NPC 对话显示剧情文本
- **商店界面**：展示可购买物品和玩家金币
- **任务追踪**：显示当前任务目标和进度
- **设置菜单**：音效、画质、控制、账号等设置

---

## 十、音效与表现系统

### 24. 音效管理

**功能描述：**
- **战斗音效**：攻击命中、受击惨叫、技能释放音效
- **环境音效**：背景音乐、风声、水流等环境音
- **UI 音效**：按钮点击、物品拾取、升级提示音

**音频资源管理：**
- 使用 AudioStreamPlayer 播放音效
- 音效池预加载常用音效
- 音量分组控制（BGM/SFX/UI）

---

### 25. 特效系统

**功能描述：**
- **技能特效**：法术飞弹、爆炸、光环等粒子效果
- **击中反馈**：受击闪白、击退、飙血效果
- **掉落光柱**：传奇物品落地时的醒目光柱

**特效实现：**
- 使用 Godot 的 GPUParticles2D/3D 创建粒子系统
- 使用 Shader 实现特殊视觉效果
- 对象池管理特效实例避免性能问题

## 💡 模块选择建议

### 方案 A：精简版（总耗时：3-4 周）
**适合：** 快速验证想法，制作 MVP（最小可行产品）

**包含模块：**
- 第一阶段全部（核心战斗）
- 第三阶段全部（挂机功能）
- 简化的属性系统（只有攻击/生命/防御）
- 简化的装备系统（只有武器和衣服）

---

**适合：** 完整的单机刷刷刷体验

**包含模块：**
- 第一阶段 + 第二阶段 + 第三阶段全部
- 简化的任务系统（只有击杀任务）
- 基础存档系统

**适合：** 长期运营的小游戏项目

**包含模块：**
- 全部五个阶段
- 完整的技能和装备系统
- 丰富的任务和成就
