# 05-暗黑挂机游戏 - 数值公式参考表 (2D 版)

## 📋 文档概述

本文档汇总了游戏开发中所需的所有核心数值公式，包括角色属性、战斗计算、装备生成、经济系统、挂机效率等各个方面。所有公式都经过平衡性考虑，可直接用于实际开发。

**适用版本：** 2D 俯视角 ARPG
**引擎：** Godot 4.x

## 一、角色属性公式

### 1.1 经验值与等级

#### 升级所需经验

```
EXP_needed = floor(Base_EXP × Level^Exponent)

参数推荐:
Base_EXP = 100        # 基础经验
Exponent = 1.5        # 经验指数
floor() = 向下取整

示例计算:
Level 1 → 2: 100 × 1^1.5 = 100 EXP
Level 2 → 3: 100 × 2^1.5 ≈ 283 EXP
Level 5 → 6: 100 × 5^1.5 ≈ 1118 EXP
Level 10 → 11: 100 × 10^1.5 ≈ 3162 EXP
Level 20 → 21: 100 × 20^1.5 ≈ 8944 EXP
Level 50 → 51: 100 × 50^1.5 ≈ 35355 EXP
Level 80 → 81: 100 × 80^1.5 ≈ 71554 EXP
```

#### 累计经验公式

```
Total_EXP(Level) = Σ(EXP_needed for each level)

近似公式:
Total_EXP ≈ Base_EXP × (Level^(Exponent+1)) / (Exponent+1)

快速查表:
Level 10: 累计约 20,000 EXP
Level 20: 累计约 120,000 EXP
Level 30: 累计约 350,000 EXP
Level 40: 累计约 750,000 EXP
Level 50: 累计约 1,400,000 EXP
Level 60: 累计约 2,300,000 EXP
Level 70: 累计约 3,500,000 EXP
Level 80: 累计约 5,000,000 EXP
```

### 1.2 生命值计算

#### 最大生命值

```
Max_HP = Base_HP + (Vitality × HP_Per_Vit) + Equipment_Bonus + (Level × Growth_HP)

参数推荐:
Base_HP = 50              # 基础生命值
HP_Per_Vit = 10           # 每点体力增加的生命
Growth_HP = 2             # 每级成长生命

示例计算:
Level 1, Vit=10, 无装备:
Max_HP = 50 + (10 × 10) + 0 + (1 × 2) = 50 + 100 + 2 = 152

Level 10, Vit=50, 装备 +100 HP:
Max_HP = 50 + (50 × 10) + 100 + (10 × 2) = 50 + 500 + 100 + 20 = 670

Level 50, Vit=200, 装备 +500 HP:
Max_HP = 50 + (200 × 10) + 500 + (50 × 2) = 50 + 2000 + 500 + 100 = 2650

Level 80, Vit=400, 装备 +1000 HP:
Max_HP = 50 + (400 × 10) + 1000 + (80 × 2) = 50 + 4000 + 1000 + 160 = 5210
```

#### 生命回复

```
HP_Regen_Per_Sec = Base_Regen + (Vitality × Regen_Per_Vit) + Equipment_Regen

参数推荐:
Base_Regen = 1.0          # 基础每秒回复
Regen_Per_Vit = 0.05      # 每点体力增加的回血

战斗状态：× 0.33（降低为三分之一）
脱战状态：× 1.0（正常回复）

示例:
Vit=50, 无装备:
HP_Regen = 1.0 + (50 × 0.05) = 1.0 + 2.5 = 3.5 HP/s

脱战 5 秒后回复量:
3.5 × 5 = 17.5 HP
```

### 1.3 法力值计算

#### 最大法力值

```
Max_Mana = Base_Mana + (Intelligence × Mana_Per_Int) + Equipment_Bonus + (Level × Growth_Mana)

参数推荐:
Base_Mana = 20            # 基础法力值
Mana_Per_Int = 15         # 每点智力增加的法力
Growth_Mana = 1           # 每级成长法力

示例计算:
Level 1, Int=10, 无装备:
Max_Mana = 20 + (10 × 15) + 0 + (1 × 1) = 20 + 150 + 1 = 171

Level 10, Int=50, 装备 +100 Mana:
Max_Mana = 20 + (50 × 15) + 100 + (10 × 1) = 20 + 750 + 100 + 10 = 880

Level 50, Int=200, 装备 +500 Mana:
Max_Mana = 20 + (200 × 15) + 500 + (50 × 1) = 20 + 3000 + 500 + 50 = 3570
```

#### 法力回复

```
Mana_Regen_Per_Sec = Base_Mana_Regen + (Intelligence × Regen_Per_Int) + Equipment_Regen

参数推荐:
Base_Mana_Regen = 2.0     # 基础每秒回蓝
Regen_Per_Int = 0.1       # 每点智力增加的回蓝

施法状态：无法自然回复（仅靠药水）
脱战状态：× 1.5（加速回复）

示例:
Int=50, 无装备:
Mana_Regen = 2.0 + (50 × 0.1) = 2.0 + 5.0 = 7.0 MP/s

脱战 10 秒后回复量:
7.0 × 1.5 × 10 = 105 MP
```

### 1.4 基础属性衍生

#### 力量衍生物

```
Physical_Damage = Strength × Str_To_Dmg

参数推荐:
Str_To_Dmg = 0.8 ~ 1.2    # 根据职业调整

职业系数:
Warrior(战士): 1.2
Mage(法师): 0.5
Archer(猎人): 0.8
Necromancer(死灵): 0.6

示例:
战士，力量=100:
物理攻击 = 100 × 1.2 = 120
```

#### 敏捷衍生物

```
Crit_Rate = Base_Crit + (Agility × Agi_To_Crit) + Equipment_Crit
Attack_Speed = Base_Atk_Speed × (1 + (Agility × Agi_To_ASPD) + Equipment_ASPD)

参数推荐:
Base_Crit = 0.05          # 5% 基础暴击率
Agi_To_Crit = 0.002       # 每点敏捷增加 0.2% 暴击
Base_Atk_Speed = 1.0      # 基础攻速（次/秒）
Agi_To_ASPD = 0.003       # 每点敏捷增加 0.3% 攻速

上限限制:
Crit_Rate_Max = 0.75      # 最高 75% 暴击率
ASPD_Max = 2.0            # 最高为基础攻速的 200%

示例:
敏捷=100, 装备 +10% 暴击:
暴击率 = 0.05 + (100 × 0.002) + 0.10 = 0.05 + 0.20 + 0.10 = 0.35 = 35%

攻速 = 1.0 × (1 + (100 × 0.003) + 0.20) = 1.0 × (1 + 0.30 + 0.20) = 1.5 次/秒
```

#### 智力衍生物

```
Spell_Power = Intelligence × Int_To_SP
Elemental_Resist = Intelligence × Int_To_Resist

参数推荐:
Int_To_SP = 1.0           # 智力转法术强度
Int_To_Resist = 0.1       # 每点智力增加 0.1% 元素抗性

示例:
智力=150:
法术强度 = 150 × 1.0 = 150
全元素抗性 = 150 × 0.1 = 15%
```

#### 体力衍生物

```
Armor = Base_Armor + (Vitality × Vit_To_Armor) + Equipment_Armor
Block_Rate = Base_Block + (Vitality × Vit_To_Block) + Equipment_Block

参数推荐:
Base_Armor = 10           # 基础护甲
Vit_To_Armor = 0.5        # 每点体力增加 0.5 护甲
Base_Block = 0.03         # 3% 基础格挡率
Vit_To_Block = 0.001      # 每点体力增加 0.1% 格挡

上限限制:
Block_Rate_Max = 0.60     # 最高 60% 格挡率

示例:
体力=100, 装备 +50 护甲:
护甲 = 10 + (100 × 0.5) + 50 = 10 + 50 + 50 = 110

格挡率 = 0.03 + (100 × 0.001) = 0.03 + 0.10 = 0.13 = 13%
```

## 二、战斗计算公式

### 2.1 物理伤害计算

#### 基础伤害

```
Base_Damage = Weapon_DPS × Skill_Multiplier + Attribute_Bonus

Weapon_DPS = (Weapon_Min_Dmg + Weapon_Max_Dmg) / 2 / Weapon_Speed

参数说明:
Weapon_Min_Dmg: 武器最小伤害
Weapon_Max_Dmg: 武器最大伤害
Weapon_Speed: 武器攻速（秒/次）
Skill_Multiplier: 技能倍率（默认 1.0）
Attribute_Bonus: 属性加成（力量 × 系数）

示例:
长剑：伤害 30-50，攻速 1.2 次/秒
技能：重击（倍率 1.5）
力量：100

Weapon_DPS = (30 + 50) / 2 / 1.2 = 40 / 1.2 ≈ 33.3
Attribute_Bonus = 100 × 1.2 = 120
Base_Damage = 33.3 × 1.5 + 120 = 50 + 120 = 170
```

#### 最终物理伤害

```
Final_Physical_Damage = Base_Damage × (1 + Damage_Bonus%) 
                       × Armor_Reduction × Location_Multiplier

Armor_Reduction = 1 - (Armor / (Armor + K × Attacker_Level))

参数推荐:
K = 1000                # 护甲系数
Location_Multiplier:
  - 头部：1.5x (爆头)
  - 背部：1.3x (背刺)
  - 正面：1.0x (普通)

下限限制:
Armor_Reduction_Min = 0.25    # 最低造成 25% 伤害（75% 减伤）

示例:
攻击方 80 级，基础伤害 500
防御方护甲 600

Armor_Reduction = 1 - (600 / (600 + 1000 × 80))
               = 1 - (600 / 80600)
               = 1 - 0.0074
               ≈ 0.9926

Final_Damage = 500 × 0.9926 ≈ 496
```

#### 护甲减伤百分比

```
Damage_Reduction_% = Armor / (Armor + 1000 × Attacker_Level) × 100%

快速查表（攻击方 80 级）:
Armor=100:  减伤 = 100/(100+80000) ≈ 0.12%
Armor=500:  减伤 = 500/(500+80000) ≈ 0.62%
Armor=1000: 减伤 = 1000/(1000+80000) ≈ 1.23%
Armor=2000: 减伤 = 2000/(2000+80000) ≈ 2.44%
Armor=5000: 减伤 = 5000/(5000+80000) ≈ 5.88%
Armor=10000:减伤 = 10000/(10000+80000) ≈ 11.11%
```

### 2.2 元素伤害计算

#### 基础元素伤害

```
Base_Elemental_Damage = Spell_Power × Skill_Multiplier × Element_Bonus

参数推荐:
Element_Bonus:
  - 火焰：1.1x (持续伤害)
  - 冰霜：0.9x (附带减速)
  - 闪电：1.0x (连锁效果)
  - 毒素：0.8x (叠加 DOT)

示例:
法术强度=200, 火球术倍率=2.0
Base_Fire_Damage = 200 × 2.0 × 1.1 = 440
```

#### 元素抗性减免

```
Effective_Resistance = Target_Resistance - Penetration
Element_Reduction = Effective_Res / (Effective_Res + 5 × Attacker_Level)
Final_Element_Damage = Base_Elemental × (1 - Element_Reduction)

参数说明:
Target_Resistance: 目标元素抗性值
Penetration: 攻击方穿透值
下限：Effective_Res >= 0

上限限制:
Element_Reduction_Max = 0.75    # 最高 75% 减伤

示例:
目标火焰抗性=100, 攻击方穿透=30, 攻击方 80 级
基础火焰伤害=500

Effective_Res = 100 - 30 = 70
Element_Reduction = 70 / (70 + 5 × 80) = 70 / 470 ≈ 0.1489
Final_Damage = 500 × (1 - 0.1489) = 500 × 0.8511 ≈ 426
```

### 2.3 暴击与命中

#### 暴击率

```
Crit_Rate = Base_Crit + (Agility × 0.002) + Equipment_Crit + Skill_Crit

参数推荐:
Base_Crit = 0.05          # 5% 基础暴击
上限：Crit_Rate_Max = 0.75    # 75% 上限

示例:
敏捷=150, 装备 +15% 暴击，技能 +5% 暴击
Crit_Rate = 0.05 + (150 × 0.002) + 0.15 + 0.05
          = 0.05 + 0.30 + 0.15 + 0.05 = 0.55 = 55%
```

#### 暴击伤害

```
Crit_Damage = Base_Damage × Crit_Multiplier

Crit_Multiplier = Base_Crit_Mult + Equipment_Crit_Dmg

参数推荐:
Base_Crit_Mult = 1.5      # 基础 150% 暴击伤害
无上限

示例:
基础伤害=1000, 装备 +50% 暴伤
Crit_Mult = 1.5 + 0.5 = 2.0
Crit_Damage = 1000 × 2.0 = 2000
```

#### 期望伤害（DPS 计算）

```
Expected_Damage = Base_Damage × (1 + Crit_Rate × (Crit_Mult - 1))

示例:
基础伤害=1000, 暴击率 40%, 暴伤 200%
Expected = 1000 × (1 + 0.40 × (2.0 - 1))
         = 1000 × (1 + 0.40)
         = 1000 × 1.40 = 1400

平均每次攻击期望伤害为 1400
```

#### 命中率

```
Hit_Rate = Base_Hit + (Attacker_Accuracy - Defender_Dodge) / 100

参数推荐:
Base_Hit = 0.95           # 95% 基础命中
下限：Hit_Rate_Min = 0.50     # 最低 50% 命中
上限：Hit_Rate_Max = 1.00     # 最高 100% 命中

闪避成功：伤害 = 0

示例:
攻击方命中=80, 防御方闪避=60
Hit_Rate = 0.95 + (80 - 60) / 100 = 0.95 + 0.20 = 1.15
由于超过上限，实际命中率 = 100%
```

### 2.4 攻速与冷却

#### 攻击速度

```
Actual_Atk_Speed = Base_Atk_Speed × (1 + ASPD_Bonus%)
Attack_Interval = 1 / Actual_Atk_Speed

参数推荐:
ASPD_Bonus_Max = 1.0      # 最高 +100% 攻速

示例:
基础攻速=1.2 次/秒，装备 +30% 攻速
Actual = 1.2 × (1 + 0.30) = 1.56 次/秒
Interval = 1 / 1.56 ≈ 0.64 秒/次
```

#### 施法速度

```
Actual_Cast_Time = Base_Cast_Time / (1 + Cast_Speed_Bonus%)

参数推荐:
Cast_Speed_Max = 1.0      # 最高 +100% 施法速度（减少 50% 时间）

示例:
火球术基础施法=1.0 秒，施法速度 +50%
Actual = 1.0 / (1 + 0.50) = 1.0 / 1.5 ≈ 0.67 秒
```

#### 冷却缩减

```
Actual_CD = Base_CD × (1 - CDR%)

参数推荐:
CDR_Max = 0.50            # 最高 50% 冷却缩减

示例:
技能基础 CD=10 秒，冷却缩减 30%
Actual = 10 × (1 - 0.30) = 10 × 0.70 = 7 秒
```

## 三、装备系统公式

### 3.1 物品等级与属性区间

#### 武器 DPS 区间

```
Weapon_Min_DPS = iLevel × Coeff_A
Weapon_Max_DPS = iLevel × Coeff_B

系数推荐:
单手武器:
  Coeff_A = 1.5
  Coeff_B = 2.5

双手武器:
  Coeff_A = 2.0
  Coeff_B = 3.5

法杖:
  Coeff_A = 1.8
  Coeff_B = 3.0

示例 (iLevel=80, 单手剑):
Min_DPS = 80 × 1.5 = 120
Max_DPS = 80 × 2.5 = 200
武器伤害范围：120-200
```

#### 防具护甲值

```
Armor = iLevel × Base_Coeff × Slot_Coeff

系数推荐:
Base_Coeff = 3.0          # 基础系数

Slot_Coeff:
  衣服 (Chest): 1.0
  头盔 (Helm): 0.7
  护腿 (Legs): 0.8
  肩膀 (Shoulder): 0.6
  鞋子 (Boots): 0.5
  手套 (Gloves): 0.5

示例 (iLevel=80):
衣服护甲 = 80 × 3.0 × 1.0 = 240
头盔护甲 = 80 × 3.0 × 0.7 = 168
```

#### 饰品主属性

```
Main_Stat = iLevel × Stat_Coeff

系数推荐:
戒指/项链:
  Stat_Coeff = 3 ~ 5

示例 (iLevel=80):
主属性范围 = 80 × (3 ~ 5) = 240 ~ 400
```

### 3.4 装备价格计算

#### 基础价格

```
Base_Price = iLevel × Price_Coeff

系数推荐:
武器：Price_Coeff = 15
防具：Price_Coeff = 10
饰品：Price_Coeff = 12
消耗品：Price_Coeff = 5
```

#### 品质系数

```
Quality_Multiplier:
  Common (白色):   1.0
  Magic (蓝色):    2.5
  Rare (黄色):     6.0
  Legendary (橙色): 15.0
```

#### 词缀加成

```
Affix_Bonus = 1 + (Affix_Count × 0.20)

每条词缀增加 20% 价格
```

#### 买卖价格

```
购买价格 = Base_Price × Quality_Mult × Affix_Bonus
出售价格 = 购买价格 × 0.5  (半价回收)

示例:
iLevel=80 的稀有胸甲，4 条词缀
Base = 80 × 10 = 800
Quality = 6.0
Affix = 1 + (4 × 0.20) = 1.80

购买价 = 800 × 6.0 × 1.80 = 8640 金币
出售价 = 8640 × 0.5 = 4320 金币
```

## 四、经济与掉落公式

### 4.1 金币掉落

#### 怪物金币

```
Gold_Drop = Base_Gold × (1 + Gold_Find%) × Elite_Bonus

参数推荐:
Base_Gold = Monster_Level × Coeff
Coeff = 8                 # 金币系数

Elite_Bonus:
  普通怪：1.0x
  精英怪：3.0x
  BOSS: 10.0x

示例:
80 级普通怪，玩家 +20% 金币获取
Base = 80 × 8 = 640
Gold = 640 × (1 + 0.20) × 1.0 = 768

80 级 BOSS:
Gold = 640 × (1 + 0.20) × 10.0 = 7680
```

#### 区域金币系数

```
Difficulty_Multiplier:
  普通难度：1.0x
  噩梦难度：1.5x
  地狱难度：2.0x
```

### 4.2 经验值获取

#### 杀怪经验

```
Mob_EXP = Base_EXP × Level_Modifier × Group_Modifier × Difficulty_Modifier

Base_EXP = Monster_Level × EXP_Coeff
EXP_Coeff = 25              # 经验系数

Level_Modifier:
  Player_Level == Mob_Level: 1.0
  Player_Level < Mob_Level: 1.0 + (Diff × 0.10), max 1.5
  Player_Level > Mob_Level: 1.0 - (Diff × 0.20), min 0.2

Group_Modifier:
  单人：1.0
  2 人组队：0.8 (每人)
  3 人组队：0.7 (每人)
  4 人组队：0.6 (每人)

示例:
玩家 80 级，击杀 85 级怪物，单人
Base = 85 × 25 = 2125
Level_Diff = 85 - 80 = 5
Level_Mod = 1.0 + (5 × 0.10) = 1.5
EXP = 2125 × 1.5 × 1.0 = 3187.5 ≈ 3188
```

#### 任务经验

```
Quest_EXP = Quest_Level × Base_Reward × Completion_Bonus

参数推荐:
Base_Reward = 50
Completion_Bonus:
  主线任务：3.0x
  支线任务：1.5x
  每日任务：1.0x

示例:
80 级主线任务
EXP = 80 × 50 × 3.0 = 12000
```

### 4.3 物品掉落率

#### 基础掉落概率

```
Drop_Chance = Base_Drop_Rate × Drop_Find% × Luck_Bonus

参数推荐:
Base_Drop_Rate:
  药水：0.30 (30%)
  装备：0.15 (15%)
  宝石：0.05 (5%)
  传奇：0.01 (1%)

Luck_Bonus:
  幸运属性每点 +0.5% 掉率

示例:
玩家有 50 点幸运，击杀 BOSS
传奇基础掉率=1%
Drop_Chance = 0.01 × (1 + 0.50) × (1 + 50×0.005)
            = 0.01 × 1.50 × 1.25
            = 0.01875 = 1.875%
```

#### 保底机制

```
Pity_System:
  每击杀 N 个 BOSS 未掉落传奇，增加概率

增加公式:
Increased_Rate = (Kills_Without_Legendary / Pity_Threshold) × Base_Rate

参数推荐:
Pity_Threshold = 50       # 50 次保底
增加上限：2.0x

示例:
已击杀 30 个 BOSS 未掉落传奇
基础掉率=1%
Increased = (30 / 50) × 1% = 0.6%
实际掉率 = 1% + 0.6% = 1.6%
```

## 五、挂机效率公式

### 5.1 DPS 估算

#### 理论 DPS

```
Theoretical_DPS = (Skill_Damage / Skill_CD) + Auto_Attack_DPS

Auto_Attack_DPS = Weapon_DPS × Hit_Rate

示例:
技能：伤害 500, CD 5 秒
武器 DPS: 100
命中率：95%

Skill_DPS = 500 / 5 = 100
Auto_DPS = 100 × 0.95 = 95
Theoretical = 100 + 95 = 195 DPS
```

#### 持续 DPS（考虑法力限制）

```
Sustainable_DPS = Total_Damage / (Attack_Time + Regen_Time)

Regen_Time = (Total_Mana_Cost - Natural_Regen) / Potion_Regen × Potion_CD

参数说明:
Total_Damage: 一轮输出的总伤害
Attack_Time: 输出时间（秒）
Natural_Regen: 自然回复的法力
Potion_Regen: 药水回复量
Potion_CD: 喝药 CD（秒）

示例:
一轮输出耗时 10 秒，造成伤害 2000，消耗法力 500
自然回复 100MP，药水回复 300MP，喝药 CD 2 秒

Regen_Time = (500 - 100) / 300 × 2 = 400/300 × 2 ≈ 2.67 秒
Sustainable = 2000 / (10 + 2.67) = 2000 / 12.67 ≈ 158 DPS
```

---

### 5.2 离线收益

#### 基准收益

```
Hourly_Gain = DPS × Mob_HP × Spawn_Rate × 3600

参数说明:
DPS: 玩家秒伤
Mob_HP: 怪物平均血量
Spawn_Rate: 怪物刷新率（只/秒）
3600: 一小时秒数

示例:
DPS=200, 怪物平均 HP=400, 刷新率=0.5 只/秒
Hourly = 200 × 400 × 0.5 × 3600 = 144,000,000 伤害转化

假设每 400HP=1EXP, 每 1000HP=1 金币
EXP/Hour = 144,000,000 / 400 = 360,000 EXP
Gold/Hour = 144,000,000 / 1000 = 144,000 金币
```

#### 衰减系数

```
Offline_Gain = Hourly_Gain × Hours × Decay_Coefficient

Decay_Coefficient:
  0-4 小时：1.0
  4-8 小时：每小时 -0.10
  8-12 小时：每小时 -0.20
  >12 小时：0（不再累积）

分段计算:
前 4 小时：Hours_1 × 1.0
4-8 小时：Hours_2 × (1.0 - 0.10 × (Hours_2))
8-12 小时：Hours_3 × (0.60 - 0.20 × (Hours_3 - 8))

示例:
离线 10 小时，每小时收益 100k EXP

前 4 小时：4 × 1.0 = 4.0
4-8 小时：4 × (1.0 - 0.10×4) = 4 × 0.6 = 2.4
8-10 小时：2 × (0.60 - 0.20×2) = 2 × 0.2 = 0.4
Total_Coeff = 4.0 + 2.4 + 0.4 = 6.8

Offline_Gain = 100k × 6.8 = 680k EXP
（而非 1000k，体现了衰减）
```

#### 收益上限

```
Max_Offline_Gain = Online_Gain_Per_Hour × Cap_Hours

参数推荐:
Cap_Hours = 1             # 最多不超过在线 1 小时的收益

目的：防止离线收益过高影响游戏平衡
```

### 6.3 玩家战力评估

#### 综合战力评分

```
Combat_Power = DPS_Score + Survival_Score + Utility_Score

DPS_Score = (DPS × 10) + (Crit_Rate × 100) + (ASPD × 50)
Survival_Score = (EHP / 100) + (Resist_Total / 10) + (HP_Regen × 5)
Utility_Score = (Move_Speed × 2) + (CDR × 100) + (Gold_Find × 50)

EHP (有效生命值) = HP / (1 - Damage_Reduction%)

示例:
玩家属性:
DPS=200, 暴击=35%, 攻速=1.5 次/秒
HP=3000, 减伤=40%, 全抗=50, 回血=20/s
移速=120%, CDR=30%, 金币=50%

DPS = (200×10) + (0.35×100) + (1.5×50) = 2000 + 35 + 75 = 2110
EHP = 3000 / (1 - 0.40) = 3000 / 0.60 = 5000
生存 = (5000/100) + (50/10) + (20×5) = 50 + 5 + 100 = 155
功能 = (120×2) + (0.30×100) + (50×50) = 240 + 30 + 2500 = 2770

总战力 = 2110 + 155 + 2770 = 5035
```

## 八、数值平衡建议

### 8.1 测试方法

```
1. DPS 测试
   - 使用标准装备测试对木桩的输出

   - 记录金币收入和支出

   - 实际练级到满级
   - 记录每级所需时间
   - 调整经验公式确保节奏合理
```
