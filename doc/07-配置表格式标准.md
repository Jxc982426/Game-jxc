# 07-配置表格式标准

## 📋 文档概述

本文档定义了游戏开发中使用的所有配置表的格式标准，包括 JSON 数据结构、字段命名规范、ID 编码规则等。所有开发人员必须遵循此标准以确保数据一致性。

**创建日期：** 2026-04-02  
**适用版本：** v1.0

---

## 一、通用命名规范

### 1.1 字段命名规则

```
命名风格：snake_case（小写 + 下划线）
示例：item_id, max_health, damage_multiplier

布尔类型：is_, has_, can_ 前缀
示例：is_active, has_effect, can_stack

数值类型：明确单位
示例：damage_value, health_percent, cooldown_seconds

数组类型：复数形式
示例：skills, affixes, drop_items
```

### 1.2 ID 编码规范

```
格式：[类别前缀]_[子类别]_[序号]

物品 ID: item_[type]_[id]
- item_wep_001 (武器 - 剑 -001)
- item_arm_002 (防具 - 衣服 -002)
- item_pot_003 (消耗品 - 药水 -003)

敌人 ID: enemy_[zone]_[id]
- enemy_town_001 (城镇区域 -001)
- enemy_dun1_002 (地下城 1-002)
- enemy_boss_001 (BOSS-001)

技能 ID: skill_[class]_[id]
- skill_war_001 (战士技能 -001)
- skill_mag_002 (法师技能 -002)
- skill_nec_003 (死灵技能 -003)

任务 ID: quest_[type]_[id]
- quest_main_001 (主线任务 -001)
- quest_side_002 (支线任务 -002)
- quest_daily_003 (每日任务 -003)

地图 ID: map_[zone]_[id]
- map_town_001 (城镇 -001)
- map_dun1_001 (地下城 1-001)
```

### 2.2 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| item_id | string | ✅ | 唯一物品 ID |
| item_name | string | ✅ | 物品名称 |
| item_type | string | ✅ | 大类：weapon/armor/consumable/material |
| subtype | string | ✅ | 子类：sword/bow/chest/potion |
| quality | int | ✅ | 品质：0=普通，1=魔法，2=稀有，3=传奇 |
| item_level | int | ✅ | 物品等级 |
| base_stats | object | ❌ | 基础属性（装备类需要） |
| effects | object | ❌ | 效果（消耗品需要） |
| requirements | object | ❌ | 装备/使用要求 |
| affixes | object | ❌ | 词缀池配置 |
| price | object | ✅ | 价格配置 |
| icon | string | ✅ | 图标路径 |
| description | string | ✅ | 物品描述 |
| max_stack | int | ✅ | 最大堆叠数 |

## 八、配置表验证工具

### 8.1 JSON Schema 验证

```python
# validate_config.py
import json
import jsonschema

def validate_items_config(data):
    schema = {
        "type": "object",
        "properties": {
            "items": {
                "type": "array",
                "items": {
                    "type": "object",
                    "required": ["item_id", "item_name", "item_type", "quality"],
                    "properties": {
                        "item_id": {"type": "string", "pattern": "^item_[a-z]+_[0-9]+$"},
                        "quality": {"type": "integer", "minimum": 0, "maximum": 3},
                        "max_stack": {"type": "integer", "minimum": 1}
                    }
                }
            }
        }
    }
    jsonschema.validate(data, schema)

# 批量验证所有配置
def validate_all_configs():
    configs = [
        ("items.json", validate_items_config),
        ("enemies.json", validate_enemies_config),
        ("skills.json", validate_skills_config)
    ]
    
    for config_file, validator in configs:
        with open(f"config/{config_file}", 'r', encoding='utf-8') as f:
            data = json.load(f)
            validator(data)
            print(f"✅ {config_file} 验证通过")
```
