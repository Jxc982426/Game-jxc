# 08-数值平衡测试报告

## 📋 文档概述

本文档记录了游戏数值系统的实际测试数据、平衡性分析和调整记录。所有数据基于 Godot 4.x 引擎的实际运行测试。

**测试版本：** v1.0  
**测试日期：** 2026-04-02  
**测试人员：** 开发团队

---

## 一、DPS 测试报告

### 1.1 测试环境

```
测试等级：80 级
装备水平：标准史诗装（iLevel 80）
目标假人：HP 1000000, 护甲 0
测试时长：30 秒
重复次数：5 次取平均值
```

### 1.2 各职业 DPS 数据

#### 战士（双手武器流）

```
技能循环：
1. 重击 (CD 好就用)
2. 顺劈斩 (填充技能)
3. 普通攻击 (CD 间隙)

理论计算:
- 重击 DPS: 500 / 3.0 = 167
- 顺劈 DPS: 200 / 1.5 = 133
- 普攻 DPS: 150 × 1.2 = 180
- 理论总 DPS: 480

实际测试结果:
| 测试次数 | 总伤害 | DPS   |
|----------|--------|-------|
| 1        | 14520  | 484   |
| 2        | 14380  | 479   |
| 3        | 14610  | 487   |
| 4        | 14290  | 476   |
| 5        | 14500  | 483   |
| 平均     | 14460  | 482   |

偏差：+0.4% ✅ (可接受范围)
```

#### 法师（火法流）

```
技能循环:
1. 火球术 (主输出)
2. 烈焰风暴 (AOE)
3. 火焰冲击 (填充)

理论计算:
- 火球 DPS: 400 / 2.0 = 200
- 烈焰 DPS: 600 / 4.0 = 150
- 冲击 DPS: 150 / 1.0 = 150
- 理论总 DPS: 500

实际测试结果:
| 测试次数 | 总伤害 | DPS   |
|----------|--------|-------|
| 1        | 14100  | 470   |
| 2        | 14400  | 480   |
| 3        | 14250  | 475   |
| 4        | 14550  | 485   |
| 5        | 14100  | 470   |
| 平均     | 14280  | 476   |

偏差：-4.8% ⚠️ (需要微调)
问题：施法时间受网络延迟影响
调整：减少火球术施法时间从 1.0s → 0.9s
```

#### 猎人（射击流）

```
技能循环:
1. 瞄准射击 (CD 好就用)
2. 多重射击 (填充)
3. 自动射击

理论计算:
- 瞄准 DPS: 600 / 3.0 = 200
- 多重 DPS: 250 / 1.5 = 167
- 自动 DPS: 120 × 1.5 = 180
- 理论总 DPS: 547

实际测试结果:
平均 DPS: 520
偏差：-4.9% ⚠️
问题：移动中输出损失
调整：增加"移动施法"被动技能
```

#### 死灵（召唤流）

```
召唤物配置:
- 骷髅战士 × 4
- 骷髅法师 × 2

理论计算:
- 本体 DPS: 150
- 骷髅战士 DPS: 80 × 4 = 320
- 骷髅法师 DPS: 100 × 2 = 200
- 理论总 DPS: 670

实际测试结果:
平均 DPS: 645
偏差：-3.7% ✅
```

### 1.3 结论与调整

```
✅ 战士：平衡良好
⚠️ 法师：施法速度偏慢 → v1.1 版本调整
⚠️ 猎人：移动战损失大 → 新增移动施法天赋
✅ 死灵：召唤物流派表现稳定
```

## 九、数值敏感点分析

### 9.1 关键阈值

```
暴击率:
- 25%：质变点（4 次攻击 1 暴击）
- 50%：理想值（2 次攻击 1 暴击）
- 75%：上限（不再提升）

攻击速度:
- 1.5 次/秒：第一阈值
- 2.0 次/秒：第二阈值（手感质变）

冷却缩减:
- 30%：技能流畅
- 40%：接近上限
- 50%：上限（边际效应递减）
```

### 9.2 职业平衡要点

```
战士:
- 依赖近战，怕风筝
- 生存能力强
- 爆发中等，持续输出高

法师:
- 脆皮，需要距离控制
- 爆发高，CD 长
- 需要法力管理

猎人:
- 机动性强
- 持续输出稳定
- 依赖装备

死灵:
- 召唤物分担伤害
- 本体脆弱
- 操作要求低
```
